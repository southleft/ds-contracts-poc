# `extract/computed/` — the production computed-capture floor

Code-side capture of **computed-style truth from real rendered components**,
fused with the static extraction's semantic layer. This module productionizes
the validated spike in `extract/computed-spike/` (see its `DESIGN.md` — the
design this module executes; the spike stays committed as the
prototype-with-receipts). Spike gate numbers it was promoted on: 99.947%
replay computed equality, 480/480 pixel pairs at the AA point, on the full
Polaris Button variant×state space.

## What a run produces (per component, in `out/<component>/`)

| artifact | what it is |
|---|---|
| `captured-truth.json` | provenance-pinned capture: browser build, viewport, writing mode, the browser's OWN enumerated longhand list (no whitelist), per-combo×state per-part computed deltas over the base capture. **Replay-sufficient** (asserted every run: reconstruction from this file byte-equals the in-memory sweep). |
| `enriched.contract.json` | the static contract enriched with minted computed bindings — schema-valid (`ContractSchema.parse`) AND generator-valid (`validateContract`), regenerated byte-reproducibly. |
| `enriched.extension.json` | everything the vocabulary cannot carry, BY NAME: code-only channels, overflow bindings, fold receipts, pseudo-element findings, structure receipts, the anatomy join. |
| `review-queue.json` | **binding contradictions** (computed truth vs static binding) with evidence + DTCG candidates — the input to the resolution workflow below. |
| `scorecard.json` | the fidelity gate: enriched contract → `emit-html` → same-browser **pixel + computed-equality vs the ORIGINAL npm package** per variant×state — % computed-equal, % pixel-perfect at threshold 0 AND the AA point, bound/minted/code-only counts, contradictions, named losses. |
| `numbers.json`, `pixel-rows.json`, `LEDGER.md` | the measured numbers (spike-comparable), per-pair pixel rows, and the human-readable delta ledger. |
| `receipts/*.png` | paired original-vs-replay screenshots (base, hover, focus, worst row). |

## Running

```bash
# harness = npm sandbox OUTSIDE the repo (verify.ts pattern):
#   npm i @shopify/polaris@13.9.5 react@18 react-dom@18 esbuild
npm run extract:computed -- --harness <dir>                    # all configured components
npm run extract:computed -- --harness <dir> --component Button # one
npm run extract:computed -- --harness <dir> --config extract/computed/configs/polaris.json
```

Needs a local Chromium (playwright cache / system Chrome /
`PLAYWRIGHT_CHROMIUM_PATH` — the visual-parity discovery convention).
Network-free at run time. The run REFUSES on library-version drift between
the harness and the config pin, and FAILS if the double-run byte-identity
self-check breaks (unstable channels are named, never hidden).

## Configuration (`configs/*.json`)

Config-driven per DESIGN §1: the **library mount recipe** (package, provider
imports/wrapper, class prefix), the **component list**, and per component the
**prop-space source** — always a static-extraction contract/proposal; the
capture tool never re-derives the API:

- `axes`: enum props that enumerate. A **defaultless enum** gets the
  `enumeration.unsetLabel` pseudo-value prepended — unset is a first-class
  point in the space (S2). Every non-axis enum prop is held at its default
  and receipted (`axesHeldFixed`).
- `stateProps`: boolean props driven as states (`disabled`) — 2-value axes
  AND state guards; interactions still run on flagged combos and any styling
  delta there is an `interaction-on-<prop>-changed` receipt.
- `triage`: named causes for expected binding contradictions (the verify.ts
  curation discipline — an untriaged contradiction is a defect).
- Enumeration policy (§1.4): **full cartesian when ∏|Aᵢ| ≤
  `cartesianLimit`** (512); above that, per-axis + pairwise with rotated
  contexts, and the certificate REFUSES ≥3-axis-interacting channels by name
  (`pairwise-inconsistent`) — pairwise can detect such channels but never
  value unvisited points.

## The pipeline

1. **CAPTURE** — real components mounted per combo inside the library's
   providers (flex mount stage: the mount-context strut receipt), states
   driven the visual-parity way (residual-pointer neutralization,
   sentinel+Tab keyboard modality, hover+mouse-down), every enumerated
   longhand read per element incl. `::before`/`::after`. Transitions stay
   ENABLED; paint channels are polled to steady state. Double full sweep →
   byte-identity REQUIRED.
2. **FUSE** —
   - **folding pass** (item 4 / spike risk #2): channels that provably track
     a source fact in EVERY capture fold into it — `currentColor`-initialized
     color channels into `color`, constant-em-ratio px channels into
     `font-size`. Receipted; before→after minted-leaf counts quoted.
   - **BOUND**: carried bindings browser-probed to canonical values and
     string-compared per combo (no tolerance). Mismatches are
     **binding-contradiction receipts** → `review-queue.json`.
   - **MINTED**: unlabeled styled channels through `core/mint-tokens.ts`
     UNCHANGED (uniform / per-axis / per-pair / refused-by-name).
     Application is S2/S3-hardened: unset-axis leaves become base bindings +
     `tokensByProp` maps; pair-with-unset carries via the reviewed emitter
     capability lift (one-placeholder map refs → compound enum-class rules);
     per-axis state refs land in root `states` (the emitters expand them);
     state×pair and state×unset set-planes are NAMED overflow (S3 residue).
   - **CODE-ONLY**: everything else lands in the extension block with a
     machine-readable reason. Nothing is dropped.
3. **REPLAY** — captured truth applied verbatim to a reconstructed DOM →
   computed re-read equality + pixel vs the original at threshold 0 AND the
   AA point (both quoted, never widened). Vocabulary-INDEPENDENT: measures
   the capture floor itself.
4. **GATE** (`scorecard.json`) — the contract-mediated instrument: enriched
   contract → `emit-html` (wrapped library tokens + minted token custom
   properties) vs the original rendering, per combo×state. Every fusion loss
   shows up here as an honest mismatch row.

## Contradiction resolution (item 3 — human-ack, never automatic)

```bash
npm run extract:computed:resolve -- --dir extract/computed/out/button \
  --apply "<id>[,<id>…]" [--to "{token.ref}"]
```

- Item ids come from `review-queue.json`. There is **no `--all`** — every
  resolution names its items (explicit-ack semantics).
- Computed-wins application: the observed value's UNIQUE DTCG candidate (or
  an explicit `--to`) rebinds the channel — at the base when every combo
  contradicts, or as a `tokensByProp` override on the single axis that
  cleanly partitions contradicting from confirming combos. No single axis →
  REFUSED by name (schema-level review, not a CLI patch). Partial acks for
  one binding site → REFUSED (a half-resolved binding would contradict
  itself).
- Output: `resolved.contract.json` (the generated `enriched.contract.json`
  is NEVER mutated — it stays byte-reproducible), `decisions.json` +
  `decisions.md` — the decisions ledger with evidence and provenance. These
  survive re-runs of the capture.
- Playground surfacing of the review queue is **named future UI**; the queue
  file is machine-readable for it. CLI-only today.

## The eval

`computed-floor-gate` (evals/run.ts) replays the COMMITTED Button
captured-truth fixture offline through `replay.ts` in real Chromium and
asserts computed re-read equality against the committed floor, plus the
enumeration certificate on a synthetic ≥3-axis interaction (refused by
name). No harness/npm sandbox needed; a missing Chromium fails by name
(`npx playwright install chromium` / `PLAYWRIGHT_CHROMIUM_PATH` — the
CERTIFICATION convention).

## Honest boundaries (named, not hidden)

- React mount adapter only (§1.1); custom elements are designed
  (`extract/adapters/cem.ts` supplies the attribute space) but not built;
  other frameworks are `mount-adapter-missing:<framework>` refusals.
- Geometry/layout-derived channels are captured but `environment-dependent`
  (font metrics) — excluded from cross-environment determinism claims and
  from fusion, by name.
- `@media`-conditional styling: the pinned sub-breakpoint viewport is
  provenance; conditional overrides surface as contradiction receipts (S7
  evidence), never silently pass.
- Inline/canvas emitters do not expand placeholder map refs (the capability
  lift covers the CSS emitters `emit-html`/`emit-react`); enriched contracts
  are consumed by the gate's HTML surface today — a named limit.
